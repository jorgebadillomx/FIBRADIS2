Eso impide considerarla completamente cerrada a nivel de producto aunque las historias estén marcadas como `done` en el tracker principal.

## Dificultades y Patrones Detectados

### 1. Cierre de historias inconsistente entre archivos

Se detectó una discrepancia entre el tracker global y el story file de la historia 2.2:

- `sprint-status.yaml` marca `2-2-home-publica-con-busqueda-global-y-layout: done`
- El story file `2-2-home-publica-con-busqueda-global-y-layout.md` sigue en `Status: review`

Esto no cambió el código entregado, pero sí debilita la confiabilidad del proceso de cierre.

### 2. La revisión encontró issues recurrentes de endurecimiento

En varias historias apareció el mismo patrón:

- manejo incompleto de nulls o estados vacíos
- detalles de responsive y overflow
- diferencias entre comportamiento en dev y en hosting real
- degradación de experiencia al faltar datos o al hidratar SSR

No son fallas de arquitectura; son señales de que las historias públicas requieren una checklist más estricta de cierre visual, accesibilidad y hosting.

### 3. Integración frontend-hosting fue más compleja de lo esperado

La Home y la ficha no se limitaron a “renderizar pantallas”. Hubo trabajo adicional en:

- fallback SPA en backend
- prerender para rutas públicas
- extracción e inyección correcta de metadata
- hidratación de React Query
- soporte de refresh/direct hit en rutas públicas

Esto sugiere que las historias públicas con SEO deben asumirse como integración full-stack, no como trabajo aislado de frontend.

## Aprendizajes Clave

1. Las rutas públicas indexables deben tratarse desde el inicio como entregables de hosting + SEO + accesibilidad, no solo como vistas.
2. El code review sí está agregando valor material: varias correcciones importantes evitaron bugs de producción en SSR, fallback y responsive.
3. El tracker global y el story file deben cerrarse juntos; cuando divergen, la épica parece terminada aunque el proceso no lo esté.
4. `tests + build` bastan para confianza técnica básica, pero no sustituyen release ni aceptación de stakeholders para declarar cierre real.
